Linux CLI Notes

A personal cheat sheet for Linux command line essentials — organised by topic with commands, descriptions, and examples. Updated as I progress through The Linux Command Line.

Navigation

  • cd – change directory
cd ..         # up one directory
cd /etc       # go to /etc
cd /usr/bin   # go to /usr/bin
cd ~          # current user's home
cd ~$USER     # specified user's home (if readable)
  • ls – list directories/files
ls -l     # long list
ls -la    # include hidden files
ls -li    # show inode numbers
  • pwd – print working directory
pwd
  • ALT/CTRL/A-z – alt and ctrl key shortcuts
CTRL-A     # Move cursor to the beginning of the line
CTRL-E     # Move cursor to the end of the line
CTRL-F/→   # Move cursor forward one character; same as right → key
CTRL-B/←   # Move cursor back one character; same as left ← key
ALT-F      # Move cursor forward one word
ALT-B      # Move cursor back one word
CTRL-L     # Clear the screen and move the cursor to the top-left
                          corner. Same as clear command.

File Control & Manipulation

  • mkdir – make new directory
mkdir NAME           # create directory
mkdir -p dir/newdir  # create nested dirs
  • touch – create new empty file 
touch FILE           # create empty file or update mtime
  • rm – create new empty file 
rm FILE              # remove file
rm -r DIR            # remove directory recursively (careful)
  • rmdir – create new empty file 
rmdir DIR            # remove empty directory
  • nano – create new empty file 
nano FILE            # open in nano (creates if missing)
  • cp – create new empty file 
cp SRC DEST          # copy file
cp -r SRCDIR DEST    # copy directory recursively
  • mv – create new empty file 
mv OLD NEW           # move/rename (does NOT edit files)

Searching

# find files/dirs by name/type starting at PATH
find PATH -name "pattern"           # by name
find PATH -type f -name "*.log"     # only files
find PATH -type d -name "conf*"     # only directories

# grep strings in files
grep "text" FILE
grep -i "text" FILE                 # ignore case
grep -r "text" DIR                  # recurse directory

Linking

ln TARGET LINKNAME            # hard link
ln -s /path/target symlink    # symbolic link

Output & Redirection

cmd > out.txt            # stdout to file (overwrite)
cmd >> out.txt           # stdout append
cmd 2> err.txt           # stderr to file (overwrite)
cmd 2>> err.txt          # stderr append
cmd &> all.txt           # stdout+stderr (bash)
# portable equivalent:
cmd > all.txt 2>&1

cmd1 | cmd2              # pipe cmd1 stdout into cmd2

# tee: write to file AND pass through
cmd | tee out.txt | cmd2
cmd | tee -a out.txt     # append

Help & Documentation

man CMD                  # open manual
whatis CMD               # one-line description

    Discovery and Exploration

    

    nmap
       
  Network exploration tool and security / port scanner

  -sO           # IP scan to check which IP protocols are supported by the host
  -sn           # Ping scan - Check for hosts only using ICMP/TCP/ARP/etc - but skip port scanning
  -sL           # List scan 
  -sT           # TCP 3-Way handshake on every port and drop the connection if established
  -sS           # TCP SYN only first step of 3-Way handshake and then RST 
  -sU           # UDP scan
  -sV           # Service version detection
  -F            # Fast scan of only first 100 ports
  -p[range]     # -p-25 (1-25) -p 80 (only port 80) -p- (all ports) -p200-2000 (ports 200 to 2000)
  -O            # OS detections
  -A            # OS, Version and other detections and additions 
  -Pn           # Force scan of all hosts including ones appearing to be down
  -T<0-5>       # Speed of scans -paranoid (0), sneaky (1), polite (2), normal (3), aggressive (4), and insane (5)
  -v(vvvv)      # verbpse output up to vvvv
  -d            # debugging output up to -d9
  -oN filename  # normal output
  -oX filename	# XML output
  -oG filename  # grep-able output
  -oA filename  # output in all major formats
  
  --min-parallelism <numprobes> and --max-parallelism <numprobes>     # minimum and maximum number of parallel probes
  --min-rate <number> and --max-rate <number>                         # minimum and maximum rate (packets/second)
  --host-timeout                                                      # maximum amount of time to wait for a target host
